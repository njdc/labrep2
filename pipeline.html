⚖️ AutoThresholdTuner
self_learner.py
  • Binary search over (min_support, min_confidence) space
  • Target: 8–25 quality rules per iteration
  • Quality = 0.50 × lift + 0.30 × conf + 0.20 × sup
  • Search range tightens each iteration as dataset grows
  • Up to 14 binary search steps logged to tuning history
🌲 FP-Tree Construction
fp_growth.py
  • Pass 1: Count item frequencies, filter by min_support
  • Pass 2: Insert sorted transactions into prefix tree
  • Compressed representation — no candidate generation
  • Header table links all nodes per item
📊 RuleStabilityTracker
self_learner.py
  • Tracks rule keys across all 3 iterations
  • Stable seen in ≥2 prior iterations
  • Emerging seen in 1 prior iteration
  • New first appearance this iteration
🏠 Homepage Ranking
recommender.py · rank_homepage()
  • Score = base + Σ(lift × support × 12) as consequent
  • + Σ(support × 6) as antecedent
  • + item_support × 4 popularity bonus
  • Surfaces items "pulled" by popular anchors first

🛒 Affinity-Based Generator
generate_all_batches()
  • 15 affinity groups per dataset (e.g., Laptop+Mouse+Keyboard = 18% base prob)
  • Basket size: random 1–5 items, weighted distribution
  • Batch multipliers simulate evolving buying patterns over time
  • Iter 1: ~667 txns  ·  Iter 2: ~1333  ·  Iter 3: ~2000 (cumulative)
🔧 Threshold Decision
tuner.tune(mine_fn)
  • Calls fp_growth() + generate_rules() at each step
  • Too few rules → lower min_support
  • Too many rules → raise min_support / min_confidence
  • Final (min_sup, min_conf) passed to full mining run
⛏ Frequent Itemset Mining
fp_growth() — recursive
  • Mines conditional pattern bases bottom-up
  • Grows frequent itemsets via conditional FP-Trees
  • Returns dict: frozenset → count
  • Top 40 itemsets sent to frontend UI
↩ feeds back
🔍 DriftDetector
self_learner.py
  • Compares rule support between consecutive iterations
  • Minor drift ≥ 8% support change
  • Major drift ≥ 20% support change
  • Drift events trigger narrower search range next iteration
🛍 Cross-Sell & Bundles
recommender.py

  • Bundle Deal lift ≥ 3.5 + conf ≥ 0.65
  • Cross-Sell lift ≥ 2.5 + conf ≥ 0.55
  • Soft Suggest lift ≥ 1.5
  • Bundle pricing includes % discount on combined price

🔄

Self-Learning Iteration Loop — Runs 3× per Dataset

  • Iter 1 Initial learning from ~667 transactions. Obvious associations emerge (e.g., Laptop → USB Hub). AutoTuner sets baseline thresholds.
  • Iter 2 Dataset grows to ~1333 transactions. Tuner refines thresholds. Stability tracker labels rules Emerging. Drift detector activates comparison baseline.
  • Iter 3 Full ~2000 transactions. Adaptive thresholds stabilize. Deep bundles discovered (e.g., Monitor → Mouse + SSD). Drift flags evolving patterns.
  • After each iteration, RuleStabilityTracker and DriftDetector outputs feed back into AutoThresholdTuner search range for the next iteration.
