# NexusCart — MBA Engine

Market Basket Analysis system with self-learning FP-Growth engine and
interactive e-commerce frontend.

## Business Scenario

**NexusCart** — an e-commerce platform with two stores:
- **Dataset A — GameTech Store**: Gaming & technology products
- **Dataset B — OfficePro Store**: Office & productivity products

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Run the MBA engine
```bash
py -3 main.py
```
This generates:
- `data/` — CSV transaction files (3 batches per dataset)
- `output/` — JSON results per iteration
- `frontend_data.js` — real computed data for the UI

### 3. Open the frontend
Open `index.html` directly in any browser.

> **Note:** If `frontend_data.js` is present (generated by step 2), the UI shows
> real Python-computed data. Without it, it falls back to built-in demo data.

## Self-Learning Mechanisms

1. **AutoThresholdTuner** — binary search for optimal min_support and min_confidence
2. **RuleStabilityTracker** — labels rules Stable / Emerging / New across iterations
3. **DriftDetector** — flags rules with >= 8% support change (Major Drift >= 20%)

## Algorithm: FP-Growth vs Apriori

| Aspect | Apriori | FP-Growth |
|---|---|---|
| Candidate generation | O(2^n) | None |
| Database scans | k (one per itemset size) | 2 |
| Memory | High (many candidates) | Low (compressed tree) |
| Dense baskets | Slow | Fast |
| Sparse baskets | OK | Equally good |

FP-Growth was chosen because e-commerce baskets are dense (avg 3-4 items)
and patterns are frequently queried. The compressed FP-Tree avoids the
exponential candidate explosion that makes Apriori impractical at scale.
